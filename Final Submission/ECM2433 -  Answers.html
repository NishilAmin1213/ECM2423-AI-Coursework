1.1

Search Problems are where we can represent a problem using a graph, and by searching this graph, we can find a path to the solution. Search problems require a search space, goal state, and a successor function. The 8-Puzzle can be translated into all of these, proving that it can be seen as a search problem. The Search Space being the possible permutations of tiles on the board. The Goal state would be a specific state where the tiles are in a specific position, Finally a successor function would be each possible move that can be made from each state, and form the next state in the graph.

1.2 (Part 1)

The A* Algorithm is used for traversing a graph or finding optimal paths. It combines the uniform cost search which gives priority to the lowest cost path and the greedy best-first search which chooses the path that seems best in the moment. The A* Algorithm is similar to Dijkstra's algorithm, however uses a heuristic to look ahead at each state and estimate the cost of the path from the next vertex to the goal. The usefulness of the algorithm is therefore dependent on the heuristic used, heuristic data, known as the best guess, is needed. The best guess must be admissible as an underestimate makes the algorithm less efficient, and an overestimate may prevent the algorithm working entirely.

A)

An appropriate solution space for this problem would be the number of possible solutions, this would be the population and in my code, I will use a population of 10,000. These would be generate randomly, or weighted.

I would use a 2D Array as the solution representation, this would be contained in a ‘board’ object to allow methods to be easily defined as well as attributes such as the fitness for the board to be recorded. A 2D Array would be a good choice as it can naturally store rows of a grid, and allow access in the form Array[row][col] in python.

B)

An appropriate fitness function would need to form a measure of the completeness of a board. I’ve decided to use a method which compares each attempt with the rules of Sudoku. This would produce a value based off the number of unique elements in each row, column and 3x3 box. A complete attempt would therefore have a fitness of 243 (81*3), and in order to standardise this to a range from 0 to 100, I will use a percentage value. A fitness would therefore represent a percentage completeness of the attempt from an average of the fitness of the row, column and 3x3 box.

C)

The chosen cross over function would randomly decide whether to cross over rows, columns or 3x3 spaces. This would then choose a point in each array and take the first section (beginning to this point) of the first array, and the second section (this point to the end) of the second array, using the first and second section it would combine these rows, columns or 3x3 spaces to form a child.

D)

A mutation operator would occur on a child with the probability of 0.2, this would consist of a random row being chosen, and within it, 2 elements being swapped, forming a slightly rearranged sudoku board.

E)

When initialising the population, I would start with a ‘start’ board. This would contain all the start values, and 0’s in the place of unknown values. I will implement 2 different methods of initialising the population.

1) The first method would consist of filling each empty tile with a random number between 1 and 8. This would be repeated until the maximum population size is reached.

2) Another method of initialising the population is one which would use more intuition. This method would assess each empty tile in the board, and generate a set of values in its row, column and 3x3 grid. Following this, if there are any unused values, it would fill the tile with a random one of these values. If there aren’t any possible values a random value between 1 and 8 would be chosen

Retrospectively, looking at the 2 initialisation methods I implemented, it became apparent that the 2nd method was a better approximation to the solution as it resulted as the start fitness being approximately 10% higher than when using the 1st method.

F)

I will implement 2 selection methods and will describe them as following:

1) The roulette method, this would randomly pick 2 parents from the population, however the random function would be weighted in favour of attempts with a greater fitness.

2) Greatest Fitness, the second method would consist of the selection of the two attempts with the greatest fitness

The replacement method I will choose will sort the population by fitness, and discard the half with a lower fitness, it will therefore create space to fill up the rest of this population with children of the two chosen parents, and the randomness of the cross over function as well as the possibility of mutation would create variation within these children

G)

Termination Criteria in my code will be a situation where the fitness of a child is 100, in this case, all the Sudoku rules have been solved and the goal has been reached. This may be optimistic, as the sense of randomness means that depending on the setup of variables, the fitness of future attempts may converge, however I feel it would be the most accurate termination criteria.
